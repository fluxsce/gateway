## 🐳 Docker 部署

### 前置要求

- 已安装 Docker（版本 20.10+）
- 已安装 Docker Compose（版本 2.0+）

### 1. 快速体验（推荐）

使用 Docker Compose 一键启动 MySQL + Redis + Gateway，适合快速体验和开发测试。

#### 步骤 1：准备配置文件

从项目的 `configs/` 目录拷贝配置文件到本地并修改：

**GitHub 配置文件目录：** [https://github.com/fluxsce/gateway/tree/main/configs](https://github.com/fluxsce/gateway/tree/main/configs)

**配置文件列表：**
- [app.yaml](https://github.com/fluxsce/gateway/blob/main/configs/app.yaml)
- [database.yaml](https://github.com/fluxsce/gateway/blob/main/configs/database.yaml)
- [gateway.yaml](https://github.com/fluxsce/gateway/blob/main/configs/gateway.yaml)
- [logger.yaml](https://github.com/fluxsce/gateway/blob/main/configs/logger.yaml)
- [web.yaml](https://github.com/fluxsce/gateway/blob/main/configs/web.yaml)

**主要修改 `database.yaml`：**
```yaml
database:
  type: mysql
  host: mysql  # Docker Compose 中的 MySQL 服务名
  port: 3306
  username: gateway
  password: gateway123
  database: gateway
```

#### 步骤 2：启动服务

```bash
# 进入 docker 目录
cd scripts/docker

# 启动所有服务
docker-compose up -d

# 查看服务状态
docker-compose ps
```

**预期输出：**
```
NAME              IMAGE                          STATUS         PORTS
gateway-mysql     mysql:8.0                      Up (healthy)   0.0.0.0:13306->3306/tcp
gateway-redis     redis:7-alpine                 Up (healthy)   0.0.0.0:16379->6379/tcp
gateway-app       datahub-images/gateway:3.1.1   Up (healthy)   0.0.0.0:18280->8080/tcp, ...
```

#### 步骤 3：访问服务

服务启动后，通过浏览器访问：

| 服务 | 地址 | 说明 |
|------|------|------|
| **Web 控制台** | http://localhost:12203/gatewayweb | 管理界面 |
| **API Gateway** | http://localhost:18280 | API 网关 |
| **隧道控制** | localhost:17000 | FRP 控制端口 |

**默认登录信息：**
- 用户名：`admin`
- 密码：`123456`

#### 步骤 4：查看日志

```bash
# 查看 Gateway 日志
docker-compose logs -f gateway

# 查看所有服务日志
docker-compose logs -f
```

#### 步骤 5：停止服务

```bash
# 停止服务
docker-compose stop

# 停止并删除容器（保留数据）
docker-compose down

# 停止并删除容器和数据（清空数据库）
docker-compose down -v
```

### 2. 生产部署

如果已有数据库和 Redis，只需部署 Gateway 容器。

#### 步骤 1：准备配置文件

从项目的 `configs/` 目录拷贝配置文件到本地并修改。

**GitHub 配置文件目录：** [https://github.com/fluxsce/gateway/tree/main/configs](https://github.com/fluxsce/gateway/tree/main/configs)

**配置文件列表：**
- [app.yaml](https://github.com/fluxsce/gateway/blob/main/configs/app.yaml)
- [database.yaml](https://github.com/fluxsce/gateway/blob/main/configs/database.yaml)
- [gateway.yaml](https://github.com/fluxsce/gateway/blob/main/configs/gateway.yaml)
- [logger.yaml](https://github.com/fluxsce/gateway/blob/main/configs/logger.yaml)
- [web.yaml](https://github.com/fluxsce/gateway/blob/main/configs/web.yaml)

**主要修改 `database.yaml`：**
```yaml
database:
  type: mysql
  host: 192.168.1.100      # 数据库 IP
  port: 3306
  username: gateway
  password: your-password  # 修改为实际密码
  database: gateway
```

#### 步骤 2：运行容器

```bash
# 运行 Gateway 容器
docker run -d \
  --name gateway \
  -p 8080:8080 \
  -p 12003:12003 \
  -p 7000:7000 \
  -v /opt/gateway/configs:/home/gateway/configs:ro \
  -v /opt/gateway/logs:/home/gateway/logs \
  -e TZ=Asia/Shanghai \
  -e GATEWAY_LOG_LEVEL=info \
  --restart=unless-stopped \
  crpi-25xt72cd1prwdj5s.cn-hangzhou.personal.cr.aliyuncs.com/datahub-images/gateway:3.1.1

# 查看日志
docker logs -f gateway

# 查看容器状态
docker ps | grep gateway
```

**参数说明：**
- `-p 8080:8080`：API Gateway 端口
- `-p 12003:12003`：Web 控制台端口
- `-p 7000:7000`：隧道控制端口
- `-v /opt/gateway/configs:/home/gateway/configs:ro`：挂载配置文件（只读，左侧为宿主机路径可自定）
- `-v /opt/gateway/logs:/home/gateway/logs`：挂载日志目录（可写）
- `--restart=unless-stopped`：容器自动重启

#### 步骤 3：访问服务

```bash
# 检查健康状态
curl http://localhost:12003/health

# 访问 Web 控制台
# http://localhost:12003/gatewayweb
# 默认用户名：admin
# 默认密码：123456
```

## ☸️ Kubernetes 部署

### 前置要求

- Kubernetes 集群 1.19+
- kubectl 命令行工具已安装并配置
- 集群中已部署 MySQL 数据库

### 1. 一键部署（推荐）

#### 步骤 1：准备配置文件

从项目的 `configs/` 目录拷贝配置文件到本地并修改。

**GitHub 配置文件目录：** [https://github.com/fluxsce/gateway/tree/main/configs](https://github.com/fluxsce/gateway/tree/main/configs)

**配置文件列表：**
- [app.yaml](https://github.com/fluxsce/gateway/blob/main/configs/app.yaml)
- [database.yaml](https://github.com/fluxsce/gateway/blob/main/configs/database.yaml)
- [gateway.yaml](https://github.com/fluxsce/gateway/blob/main/configs/gateway.yaml)
- [logger.yaml](https://github.com/fluxsce/gateway/blob/main/configs/logger.yaml)
- [web.yaml](https://github.com/fluxsce/gateway/blob/main/configs/web.yaml)

**主要修改 `database.yaml`：**
```yaml
database:
  type: mysql
  host: mysql-service.default.svc.cluster.local  # K8S 集群内的 MySQL 服务
  port: 3306
  username: gateway
  password: your-password  # 修改为实际密码
  database: gateway
```

#### 步骤 2：修改 YAML 文件（重要）

在执行部署前，需要根据实际环境修改 K8S 配置文件：

**1. 修改命名空间（如需要）**

默认使用 `gateway` 命名空间，如需修改，编辑以下文件：

```bash
# 修改 deployment.yaml
vim scripts/k8s/deployment.yaml
# 将 namespace: gateway 改为你的命名空间

# 修改 service.yaml
vim scripts/k8s/service.yaml
# 将 namespace: gateway 改为你的命名空间

# 修改 ingress.yaml（如果使用）
vim scripts/k8s/ingress.yaml
# 将 namespace: gateway 改为你的命名空间
```

**2. 修改镜像地址（如需要）**

编辑 `scripts/k8s/deployment.yaml`，确认或修改镜像地址：

```yaml
# 默认使用阿里云镜像（国内推荐）
image: crpi-25xt72cd1prwdj5s.cn-hangzhou.personal.cr.aliyuncs.com/datahub-images/gateway:3.1.1

# 或使用 Docker Hub 镜像
# image: datahub-images/gateway:3.1.1
```

**3. 修改副本数（如需要）**

编辑 `scripts/k8s/deployment.yaml`：

```yaml
spec:
  replicas: 1  # 根据实际需求修改副本数
```

#### 步骤 3：执行部署

```bash
# 一键安装（默认使用 ../../configs 配置目录，默认命名空间 gateway）
bash deploy.sh install

# 指定配置目录
bash deploy.sh install --config-dir /path/to/configs

# 指定命名空间
bash deploy.sh install --namespace my-namespace

# 同时指定配置目录和命名空间
bash deploy.sh install --config-dir /path/to/configs --namespace my-namespace
```

**安装成功后会显示访问信息：**
```
[SUCCESS] FLUX Gateway 安装完成！

[INFO] 访问信息:
[INFO]   API Gateway:  http://192.168.1.100:30001
[INFO]   Web 控制台:   http://192.168.1.100:30002/gatewayweb
[INFO]   隧道控制:     192.168.1.100:30003

[INFO] 默认登录信息:
[INFO]   用户名: admin
[INFO]   密码:   123456
```

#### 步骤 4：查看部署状态

```bash
# 查看详细状态（使用默认命名空间 gateway）
bash deploy.sh status

# 查看指定命名空间的状态
bash deploy.sh status --namespace my-namespace

# 或使用 kubectl 命令
kubectl get pods -n gateway
kubectl get svc -n gateway
kubectl get ingress -n gateway
```
