# 系统监控

在网关控制台中集中查看**已纳入监控的服务器**运行状况：先选定目标机器，再阅读身份信息与多组时序指标，用于日常巡检、容量评估与异常定位。

---

## 概述

**系统监控**面向运维与网关管理员，回答三类问题：

1. **这是哪台机器**——主机名、操作系统、架构、部署形态与访问地址。  
2. **资源是否健康**——CPU、内存、磁盘、磁盘 IO、网络在一段时间内是否平稳、是否逼近容量上限。  
3. **是否需要跟进**——结合页面上的预警区间、图例与空状态提示，判断是否要扩容、查日志或联系后端/采集侧。

登录后通常可从侧栏首项进入本页（具体菜单以当前环境为准）。

---

## 使用流程

### 1. 选择监控目标

在页面顶部的 **服务器选择** 中，从下拉列表指定一台**监控服务器**。列表由服务端根据权限与采集配置下发；若列表为空或长期无法加载，需要在网关或监控侧确认节点是否已注册、采集是否正常、当前账号是否有查看权限。

### 2. 刷新数据

在已选择服务器的前提下，使用 **刷新数据** 可重新请求当前视图依赖的数据（各图表上的「刷新」则只作用于该图表）。首次进入或切换服务器后，若出现加载动画，请等待结束后再读数。

### 3. 阅读指标

下方按区域展示多组图表；每一块均可调整**时间范围**并单独刷新（控件布局以当前版本界面为准）。建议先看 **CPU / 内存**，再看 **磁盘与磁盘 IO**，最后看 **网络** 与 **进程** 类视图，以便从「计算 → 存储 → 网络」逐层缩小范围。

---

## 服务器信息

选定服务器后，**服务器信息**区域展示该实例的静态概要，便于确认「当前曲线对应的是哪台机器」。常见项包括：

| 信息项 | 用途 |
|--------|------|
| 主机名 | 在资产与日志中定位该节点。 |
| 操作系统 | 区分 Windows、Linux、macOS 等大类。 |
| 系统版本 | 核对内核或发行版版本；过长时可悬停查看全文。 |
| 系统架构 | 如 x86_64，用于与软件包、镜像架构对齐。 |
| 服务器类型 | 物理机、虚拟机等归类，便于与容量规划策略对应。 |
| IP 地址 | 与网络策略、白名单及上下游连通性排查相关。 |

## 指标说明

### CPU 与内存

以**使用率随时间变化**为主；页面为使用率配置了**预警区间**（偏高与过高两档，以图例与曲线配色为准），便于快速发现持续打满或尖峰。若业务低峰期仍长期贴近上限，通常需要评估算力或排查异常进程。

### 磁盘与磁盘 IO

**磁盘**侧重空间占用趋势，**磁盘 IO** 侧重读写吞吐或 IOPS 类趋势。磁盘打满会导致写入失败与连锁故障；IO 长期冲高可能意味着慢查询、批量任务或大文件读写，需要结合业务窗口判断。

### 网络流量

区分上行与下行（或收发）方向，用颜色区分趋势，便于判断是否为突发流量、备份窗口或异常外联。若与其它指标同时异常，可优先排查网络策略与上游依赖。

### 进程

汇总与进程相关的监控信息，用于在主机层面观察**是否有异常占用**或与网关进程相关的波动；具体指标名称以界面为准。

## 空状态与排查

| 提示 | 含义与建议 |
|------|------------|
| 请选择一个服务器查看监控数据 | 列表有数据但未选择；先在顶部下拉框选定目标。 |
| 暂无可用的服务器 | 当前租户或账号下没有可监控节点；由管理员配置采集目标或授权。 |
| 正在加载服务器监控数据 | 数据拉取中；若长时间不结束，检查网络、网关服务与监控数据源。 |

若各图表长期无数据或数值明显不可信，在排除浏览器缓存与本地网络后，请通过运维渠道核对：采集代理是否在线、指标上报周期、以及网关与监控服务之间的连通性。
